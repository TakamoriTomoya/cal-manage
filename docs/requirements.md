# カロリー管理アプリ 要件定義書

## プロジェクト概要

カロリー計算ができるアプリケーション。

## 機能要件

### 1. カロリー登録機能

- **新規登録**
  - 写真をアップロードできる
  - タイトルを入力できる
  - カロリーを入力できる
  - その他の情報（任意）を入力できる
  - 登録日時を記録する

### 2. カロリー計算機能

- **その日のカロリー計算**
  - 選択した日付の摂取カロリーを自動計算
  - 登録したすべてのカロリーを合計して表示

### 3. 登録済みカロリーの再利用機能

- **選択して追加**
  - 過去に登録したカロリーを一覧表示
  - 登録済みのカロリーを選択して、新しい日付で追加できる
  - 同じカロリーを複数回追加できる
  - **カテゴリー管理**
    - 登録済みカロリーをカテゴリーで分類できる
    - カテゴリーでフィルタリングして表示できる
    - カテゴリーの追加・編集・削除ができる

### 4. カロリー確認機能

- **その日の摂取カロリー確認**
  - 選択した日付の摂取カロリーを表示
  - 登録したカロリーの一覧を表示
  - 各カロリーの詳細（写真、タイトル、カロリー数など）を確認できる

### 5. カレンダー表示機能

- **月単位のカレンダー表示**
  - カレンダー形式で月単位の表示
  - 各日の摂取カロリーをカレンダー上で確認できる
  - 日付をクリックして詳細を確認できる
  - **月の切り替え**
    - 前月・次月への切り替えができる
    - 任意の年月に移動できる

## 非機能要件

### データ保存

- **一時的な保存方法**
  - 認証とデータベースは後回し
  - ローカルストレージ（localStorage）を使用してデータを保存
  - ブラウザを閉じてもデータが保持される
- **写真の保存**
  - 一時的にローカルストレージに Base64 形式で保存
  - 後々外部ストレージ（クラウドストレージなど）への移行を想定した設計
  - 写真の最大サイズや形式の制限は実装時に決定

### UI/UX

- 直感的で使いやすいインターフェース
- モダンで美しいデザイン
- レスポンシブデザイン（モバイル対応）

## 決定事項

### 写真の保存方法

- **一時的な実装**: ローカルストレージに Base64 形式で保存
- **将来的な拡張**: 外部ストレージ（クラウドストレージなど）への移行を想定
- 写真の最大サイズや形式の制限は実装時に決定

### カロリーの単位

- **kcal（キロカロリー）で統一**
- すべてのカロリー表示・入力で kcal を使用

### 日付の扱い

- **日本時間（JST）で統一**
- タイムゾーンは日本時間（UTC+9）で処理

### カレンダーの表示形式

- **月単位で変更可能**
  - 前月・次月への切り替え機能
  - 任意の年月への移動機能
- 週の開始日やカロリーの表示方法は実装時に決定

### 登録済みカロリーの管理

- **カテゴリーで分類可能**
  - 登録済みカロリーをカテゴリーで分類できる
  - カテゴリーでフィルタリングして表示できる
  - カテゴリーの追加・編集・削除ができる
  - 例：朝食、昼食、夕食、間食、飲み物など
